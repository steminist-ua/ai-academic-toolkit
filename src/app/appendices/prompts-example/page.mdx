# Додаток Б. Зразок структури `prompts.md`

- Контекст завдання
- Обмеження/критерії якості
- Приклади (few-shot)
- Варіанти переформулювань
- Чек-лист верифікації результату

```markdown
Завдання: Рефакторинг класу `PaymentController`

Проблема:
клас має низьку зв'язність (low cohesion) та порушує принцип єдиної відповідальності (SRP).

Контекст для ШІ (System Prompt):
Ти — досвідчений Senior Software Engineer, експерт з SOLID та чистої архітектури. Твоє завдання — запропонувати варіанти рефакторингу коду на Python.

Промпт 1: Аналіз
Проаналізуй наданий код класу PaymentController і виділи основні порушення SRP.

Промпт 2: Рефакторинг
Запропонуй рефакторинг, розділивши клас на сервіс (PaymentService) та валідатор (PaymentValidator). Код має бути ідіоматичним для Python 3.11+.

```

**Чек-лист верифікації**

- [ ]  Нові класи дотримуються SRP.
- [ ]  Збережено публічний API контролера.
- [ ]  Код покритий unit-тестами.
