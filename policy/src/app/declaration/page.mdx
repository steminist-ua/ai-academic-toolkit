# 7. Вимоги до Декларації про використання ШІ


У кожній роботі здобувача, де використовувався ШІ, додається розділ «**Декларація про використання ШІ**», який містить:

> 💡 **Як створити декларацію швидко:** Скористайтеся генератором **RAID‑G** — він формує каркас декларації відповідно до цієї політики та прикладів. Посилання: https://steminist-ua.github.io/ai-academic-toolkit/raid-g/

1. **Інструменти/моделі:** назва, версія/дата доступу (напр., GPT-4o, GitHub Copilot).
2. **Мета використання:** для яких завдань застосовувався ШІ (напр., генерація boilerplate; рефакторинг; пояснення алгоритму).
3. **Промпти/Ланцюжки промптів:** ключові запити або посилання на файл `prompts.md` у репозиторії.
4. **Власний внесок та ступінь переробки:** що саме було змінено і чому; як згенерований результат був інтегрований у проєкт.
5. **Процес верифікації:** як перевірялася коректність та надійність результатів (тести, бенчмарки, code review, перевірка джерел).
6. **Критична рефлексія:** аналіз обмежень, упередженості (bias) або помилок моделі та як їх було мінімізовано.

- **МЕТА:** Рефакторинг модуля обробки подій; генерація unit-тестів для класу `UserService`.
